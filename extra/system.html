Modular Cesium Tactical Simulation Platform

A modular and replaceable architecture based on ORBAT Mapper 2D scenarios that recreates drone flight, Humvee movement, counter-drone weapon systems, PGA antenna simulation, and EO optical tracking in CesiumJS 3D, while exporting replay paths and events back into 2D formats.

Core Goal

Avoid monolithic code by designing vehicles, drones, weapons, sensors, maps, and input devices as independently replaceable modules.

2D ↔ 3D Integration

Import ORBAT Mapper 2D scenarios, replay them in Cesium 3D, and export route and engagement results back into 2D formats.

Real-Time Events

REST handles commands and configuration, SSE handles state broadcasting, and WebSocket can later support bidirectional control.

Device Input

Logitech G923 controls vehicle driving while Gladiator NXT EVO controls weapon pitch and yaw aiming.

2D → 3D → 2D Scenario Workflow

  1. Create a 2D tactical scenario in ORBAT Mapper.
  2. Export the scenario as JSON, GeoJSON, or MSDL.
  3. The Express REST API imports the scenario and drone flight files.
  4. The Simulation Core converts 2D coordinates into Cesium 3D entities.
  5. Cesium replays Humvees, drones, weapons, PGA antennas, and EO sensors in 3D.
  6. Replay-generated drone routes, engagement events, and sensor tracking states are recorded.
  7. Export 2D routes and event timelines for After Action Review.
  8. Review exported results in ORBAT Mapper or another 2D viewer.

Additional Design Considerations

  1. Coordinate Reference: ORBAT 2D 좌표, Cesium WGS84 좌표, 고도 기준을 하나로 정규화해야 합니다.
  2. Time Reference: 실제 시간, 시뮬레이션 시간, replay speed를 분리해야 합니다.
  3. Model Origin: GLB 모델의 forward axis, origin point, scale을 표준화해야 합니다.
  4. Input Processing Location: G923/NXT EVO 입력을 브라우저에서 받을지, Node 서버에서 받을지 결정해야 합니다.
  5. Sensor Tracking: PGA radar tracking과 EO optical tracking은 같은 target state를 공유해야 합니다.
  6. Replay Logging: 모든 replay 이벤트는 나중에 2D export와 분석을 위해 event log로 저장해야 합니다.
  7. Replaceability: 각 모듈은 공통 interface만 맞추고 내부 구현은 자유롭게 바꿀 수 있어야 합니다.
  8. Security: 외부 REST call을 받을 경우 scenario upload와 model path access를 제한해야 합니다.

# Modular Cesium Tactical Simulation Platform

## Vision
A modular tactical simulation platform that combines ORBAT Mapper 2D scenarios with CesiumJS 3D visualization, supporting drone simulation, Humvee vehicle control, counter-drone weapons, PGA antenna simulation, and EO optical tracking.

## Core Design Philosophy
- Avoid monolithic architecture
- Keep modules independently replaceable
- Separate REST, SSE, and simulation logic
- Support 2D ↔ 3D replay workflows
- Enable external integration through REST APIs
- Support interchangeable vehicle, drone, weapon, and sensor systems

## Main Technologies
- Node.js
- Express.js
- CesiumJS
- HTML / JavaScript
- Server-Sent Events (SSE)
- WebSocket (optional later)
- ORBAT Mapper
- MIL-STD-2525 symbology

## Main Simulation Components
- Humvee vehicle simulation
- Drone flight simulation
- Counter-drone weapon system
- PGA phased-array antenna simulation
- EO optical tracking system
- Timeline replay engine
- Scenario import/export engine

## Input Devices
- Logitech G923 for vehicle driving
- Gladiator NXT EVO for turret pitch/yaw control

## Architecture Principles
- server.js should remain lightweight
- Business logic must live in services/modules
- All systems communicate through clean interfaces
- External systems interact only through REST/SSE APIs

## Data Flow
ORBAT Mapper 2D
→ Scenario Import
→ Simulation Core
→ CesiumJS 3D Replay
→ Event Logging
→ 2D Export / After Action Review

## Long-Term Goal
Create a reusable and portable tactical simulation platform where modules can be replaced without redesigning the full system.
