Create Your First GitHub Repository

Start from the GitHub UI, create my-first-repository, add a main branch, then connect it to your local project from the VS Code terminal.

1

Open the "New repository" page on GitHub

Sign in to GitHub, then go directly to:

https://github.com/new

You'll land on the Create a new repository form. Required fields are marked with an asterisk (*).

2

Fill in the General section

Under General, set the Owner and Repository name:

github.com/new

Create a new repository

🟢 benchvue ▾
/
my-first-repository
my-first-repository is available.
Naming tip: use only lowercase letters, numbers, and hyphens. GitHub will tell you in real-time whether the name is available.
3

Choose visibility and initialize with a README

Scroll down and configure the rest of the repo:

  • Description (optional) — e.g. My first repository
  • Public — anyone can see it. Choose this if you want to use GitHub Pages later.
  • Check Add a README file — this is important. It creates a first commit so your main branch actually exists on GitHub.
  • Leave .gitignore and license as "None" for now.

Click the green Create repository button at the bottom.

Why the README matters: without any file, GitHub shows a bare "quick setup" screen with no branches. Adding a README gives you an immediate main branch you can clone and push to.
4

Confirm the main branch exists

After creation, you land on your new repo page at:

github.com/benchvue/my-first-repository

Look at the branch dropdown near the top-left of the file list — it should say main. This is your default branch.

If you didn't check "Add a README" in step 3: you'll see a setup page instead. Scroll down to "…or create a new repository on the command line" and run the commands it lists, which include git branch -M main to rename the default branch to main.
5

Copy the repo URL

On the repo page, click the green <> Code button. Under the HTTPS tab, copy the URL — it looks like:

https://github.com/benchvue/my-first-repository.git

You'll paste this into VS Code's terminal in the next step.

6

Open VS Code terminal and clone the repo

In VS Code, open the folder where you keep your projects. Open the terminal with Ctrl + ` (or ⌃ + ` on Mac), then run:

$ git clone https://github.com/benchvue/my-first-repository.git $ cd my-first-repository

You now have a local copy of the repo, already connected to GitHub as origin.

Verify the connection: run git remote -v — you should see the GitHub URL listed for both fetch and push.
7

Add a file, commit, and push

Open the folder in VS Code (File → Open Folder… → pick my-first-repository), create a new file (for example index.html), save it, then from the terminal:

$ git status # see what changed $ git add . # stage every change $ git commit -m "add index.html" $ git push # upload to GitHub

Refresh your GitHub repo page — the new file is there. From now on, the cycle is just edit → add → commit → push.

8

Alternative: push an existing local folder

If you already have a folder of files and want to publish it to the empty repo you just created, don't clone. Instead, inside that folder run:

$ git init $ git branch -M main $ git remote add origin https://github.com/benchvue/my-first-repository.git $ git add . $ git commit -m "first commit" $ git push -u origin main

git branch -M main renames the local default branch to main so it matches GitHub. The -u flag in the final push sets GitHub as the upstream, so later pushes can be just git push.

✓

Your first repository is live!

Visit github.com/benchvue/my-first-repository to see your code on GitHub.
